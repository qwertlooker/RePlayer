# RePlayer 对标 iONPlayer 实施文档

## 1. 文档目标
本文档用于指导另一个工程 agent 在当前仓库内，分阶段完成 RePlayer 的界面和功能优化，使其在不突破现有技术约束的前提下，更接近 iONPlayer 的产品体验。

本文档是“实施规格”，不是讨论稿。实现 agent 应按本文档执行，不再自行扩展需求范围，除非实现过程中遇到明确阻塞。

---

## 2. 实施目标

### 2.1 总目标
将当前“Win32 控件堆叠式播放器”升级为“以当前句学习为核心的英语精听工具”。

### 2.2 本次优化范围
本次仅做 v1.5 / v1.6 范围内的增强：

- 重构界面信息层级
- 强化当前句聚焦体验
- 补齐句级训练闭环
- 增加少量高价值轻量功能
- 完善状态表达、日志和测试

### 2.3 明确不做
以下内容本次禁止实现：

- 联网
- 登录
- AI 打分
- 波形图
- 视频
- 变速播放
- 播放列表
- 双语字幕系统
- 自动语音分句
- 大规模自绘 UI 框架
- 引入 Qt / Electron / .NET / Python

---

## 3. 当前工程现状

### 3.1 已有模块
当前仓库已具备以下模块和闭环能力：

- `audio/`：Media Foundation 音频播放
- `subtitles/`：SRT / LRC 解析和时间定位
- `recording/`：WASAPI 录音、WAV 输出、录音播放
- `core/`：`PlaybackCoordinator`
- `state/`：统一 `AppState`
- `ui/`：Win32 主窗口与基础控件
- `tests/`：单元测试已覆盖主要逻辑

### 3.2 当前短板
当前版本主要问题不在底层功能，而在产品化体验：

- UI 是线性按钮 + 列表布局，信息层级弱
- 当前句不够突出，学习焦点不足
- 模式状态、录音状态、可操作状态没有结构化表达
- 字幕列表承担了过多视觉职责
- 缺少句级导航和训练反馈
- UI 和状态之间仍以拼接字符串为主，不利于扩展

## 4. 实施原则

### 4.1 保持现有技术栈
必须继续使用：

- C++17
- Win32 API
- Media Foundation
- WASAPI
- CMake

### 4.2 保持模块清晰
只允许在现有模块边界内增强：

- `audio`
- `subtitles`
- `recording`
- `ui`
- `state`
- `core`

禁止把状态逻辑散回 `MainWindow`。

### 4.3 优先通用方案
优先选择对后续 v2 扩展友好的实现方式，不做一次性、局限性较强的硬编码方案。

### 4.4 保持绿色版要求
不能引入额外运行时依赖，不能破坏当前单目录可运行的打包方式。

## 5. 目标交互设计

## 5.1 新界面结构
主窗口改为四个区块：

### 顶部工具区
展示和操作全局资源：

- 打开音频按钮
- 打开字幕按钮
- 当前音频文件名
- 当前字幕文件名
- 当前播放时间 / 总时长
- 全局播放状态

### 中央视图区
作为主学习区，必须突出显示当前句：

- 当前句文本，大字号
- 当前句时间范围
- 当前句编号
- 当前句是否已有录音
- 当前模式
- 当前是否正在录音

### 训练控制区
集中放置句级学习操作：

- 播放 / 暂停 / 停止
- 上一句
- 下一句
- 播放原音
- 开始录音
- 停止录音
- 播放录音
- 模式选择
- 自动暂停秒数

### 字幕导航区
保留完整字幕列表，但降级为导航用途：

- 显示所有字幕
- 当前句高亮
- 点击跳转
- 每项带时间前缀，便于定位

---

## 5.2 状态显示要求
必须明确显示：

- 当前句
- 当前模式
- 是否正在录音
- 当前播放状态
- 当前句是否存在录音
- 当前可执行动作

禁止仅通过一个 `status_text` 传达全部状态。

---

## 5.3 交互行为要求

### 打开音频
- 成功后更新文件名、时长、状态
- 不自动清除字幕
- 如果当前有字幕，允许继续按现有句子工作，但时间同步以新音频为准

### 打开字幕
- 成功后刷新字幕列表
- 当前句立即重置并随播放位置重新计算
- 中央当前句面板同步刷新

### 点击字幕
- 跳转到该句开始时间
- 当前句面板立即更新
- 列表高亮同步

### 上一句 / 下一句
- 无字幕时按钮禁用
- 到边界时保持在首句或末句，不越界
- 导航后应执行 seek 到目标句开始时间
- 不强制自动播放，保持当前播放状态不被额外改变

### 播放原音
- 仅在有当前句时可用
- 播放当前句起止时间对应的原音片段
- 成功后状态区给出反馈

### 开始录音
- 仅在有当前句时可用
- 如果当前正在播放音频，先暂停，再开始录音
- 使用“每句一个录音，允许覆盖”的现有规则
- 若当前句已有录音，覆盖前在状态区提示，不弹确认框

### 停止录音
- 停止后立即更新“当前句已有录音”状态
- 当前句的“播放录音”按钮立刻变为可用

### 播放录音
- 仅在当前句存在录音时可用
- 无录音时必须提示用户，但不应因为正常操作频繁弹错误框
- 成功后状态区给出反馈

### 模式切换
支持：

- `Normal`
- `RepeatOne`
- `AutoPause`

要求：

- `RepeatOne > AutoPause`
- UI 必须明确显示当前模式
- 模式切换后中央区和控制区同时刷新

### 自动暂停秒数
- 允许输入整数秒
- 非法输入按 0 处理或回退为最近有效值，但实现需统一
- 默认继续使用当前的秒级配置方式

## 6. 数据与接口改造

## 6.1 AppState 改造
修改 `state/app_state.h`，新增界面驱动字段，使 UI 可以“渲染状态”，而不是拼接状态。

建议新增字段：

- `std::wstring audio_display_name`
- `std::wstring subtitle_display_name`
- `std::wstring current_sentence_text`
- `std::int64_t current_sentence_start_ms`
- `std::int64_t current_sentence_end_ms`
- `bool has_current_sentence_recording`
- `bool can_play`
- `bool can_pause`
- `bool can_stop`
- `bool can_seek`
- `bool can_go_previous_sentence`
- `bool can_go_next_sentence`
- `bool can_play_original_segment`
- `bool can_start_recording`
- `bool can_stop_recording`
- `bool can_play_recording`
- `std::wstring playback_status_text`
- `std::wstring mode_status_text`
- `std::wstring recording_status_text`
- `std::wstring hint_text`

保留现有字段：

- `audio_path`
- `subtitle_path`
- `subtitles`
- `current_subtitle_index`
- `playback_mode`
- `player_state`
- `position_ms`
- `duration_ms`
- `auto_pause_delay_ms`
- `is_recording`
- `status_text`
- `recording_by_sentence`
- `auto_pause_resume_at_ms`

处理原则：

- `status_text` 可继续保留给通用日志/提示兼容使用
- 新 UI 以结构化字段为主，不再从 `status_text` 反推状态

---

## 6.2 PlaybackCoordinator 改造
修改 `core/playback_coordinator.h/.cpp`，让 coordinator 负责完整刷新 `AppState`。

新增接口：

- `Result<void> SelectPreviousSubtitle();`
- `Result<void> SelectNextSubtitle();`

新增私有方法建议：

- `void RefreshDerivedState();`
- `void RefreshCurrentSentenceState();`
- `void RefreshActionAvailability();`
- `void RefreshStatusTexts();`

执行要求：

- 所有对外操作后都必须刷新派生状态
- `Tick()` 末尾必须刷新一次完整派生状态
- `LoadAudio`、`LoadSubtitles`、`Play`、`Pause`、`Stop`、`Seek`、`SelectSubtitle`、`StartRecording`、`StopRecording`、`PlayOriginalSegment`、`PlayRecording`、`SetPlaybackMode`、`SetAutoPauseDelayMs` 之后都要更新可视状态

### 当前句派生逻辑
当存在当前句时，必须同步以下字段：

- 当前句文本
- 当前句开始时间
- 当前句结束时间
- 当前句是否已有录音
- 是否允许播放原音
- 是否允许开始录音
- 是否允许播放录音
- 是否允许切到上一句/下一句

当不存在当前句时，上述字段重置为默认空状态。

### 录音逻辑要求
`StartRecording()` 中新增策略：

- 如果 `player_state == Playing`，先调用 `Pause()`
- 再开始录音
- 如果暂停失败，则返回错误，不进入录音
- 若录音成功，将 `is_recording = true`
- 更新录音状态文本与提示文本

### 覆盖录音策略
如果当前句已有录音：

- 允许直接覆盖
- 写日志
- 在 `hint_text` 中提示“已覆盖该句旧录音”或等价文案
- 不增加额外版本文件

### 句级导航逻辑
`SelectPreviousSubtitle()` / `SelectNextSubtitle()` 规则：

- 若无字幕，返回 `InvalidState`
- 若当前没有选中句子：
  - 上一句默认跳到首句
  - 下一句默认跳到首句
- 到边界时停在边界
- 本质仍调用 `Seek(target.start_ms)` 完成跳转

### 新增控件

- 当前句大文本标签
- 当前句时间标签

- 打开字幕

- 停止
- 进度条
- 字幕列表
- 模式下拉
- 自动暂停输入框
- 开始录音
- 停止录音
- 播放原音
- 播放录音

### 布局要求
不要求复杂自绘，但必须有明确分组和层级。

建议实现：
- 统一边距和按钮尺寸
- 中央当前句区域高度明显大于其他区域
- 使用较大字体显示当前句
- 使用较小字体显示文件信息、提示信息、句子时间
- 字幕列表放在窗口下半区或右侧固定导航区

### 渲染要求
新增统一刷新逻辑，例如：

- `SyncUi()` 中不再只更新少数字段
- 而是完整同步：
  - 文本
  - 按钮启用/禁用
  - 当前句信息
  - 模式标签
  - 录音标签
  - 列表选中项
  - 进度条位置

### 字幕列表文本格式
`PopulateSubtitleList()` 改为显示带时间前缀的文本，例如：

`00:12.340  Hello everyone.`

如果现有 `FormatTimestamp()` 不适合精确到毫秒，可新增 UI 专用格式化函数，但不要破坏已有测试依赖的旧格式。

## 7. 分阶段实施步骤

## 阶段 A：状态设计落地
目标：先把状态改对，再动 UI。

步骤：

1. 阅读 `state/app_state.h`
2. 增加结构化 UI 字段
3. 确认默认值安全
4. 不修改 UI，先只编译通过
5. 为新增字段补单元测试或在 coordinator 测试中验证默认状态

完成标准：
- 工程可编译
- 新字段存在且默认状态明确
- 未破坏现有测试

---

## 阶段 B：Coordinator 派生状态重构
目标：让 UI 只消费状态，不额外判断业务。

步骤：

1. 在 `PlaybackCoordinator` 中增加派生状态刷新方法
2. 在所有主要操作后调用刷新逻辑
3. 增加上一句 / 下一句接口
4. 增加录音前自动暂停逻辑
5. 增加录音覆盖提示和日志
6. 统一刷新：
   - 当前句信息
   - 文件显示名
   - 按钮可用性
   - 模式文案
   - 录音文案
   - 提示文案

完成标准：
- 单元测试可验证按钮状态和当前句派生状态
- `PlaybackCoordinator` 成为唯一业务状态协调点

---

## 阶段 C：UI 结构重排
目标：完成视觉信息架构重组。

步骤：

1. 修改 `MainWindow` 成员，增加新控件句柄
2. 重写 `CreateControls()`
3. 重写 `LayoutControls()`
4. 调整窗口默认尺寸，确保内容不拥挤
5. 为当前句标签设置更大的字体
6. 将字幕列表位置调整为辅助导航区
7. 增加上一句 / 下一句事件处理
8. 将模式、录音状态、提示信息分开展示

完成标准：
- 启动后主界面具有清晰分区
- 当前句是最突出信息
- 功能按钮与状态展示位置合理
- 窗口缩放时布局仍然可用

---

## 阶段 D：句级学习体验补强
目标：补齐轻量高价值能力。

步骤：

1. 接通上一句 / 下一句按钮
2. 校准无字幕时的禁用逻辑
3. 校准无录音时“播放录音”禁用逻辑
4. 校准录音中禁止重复点击“开始录音”
5. 校准录音停止后立即可播放录音
6. 校准自动暂停和单句复读在 UI 上的模式反馈
7. 校准状态提示文案，不要只显示通用 `Ready`

完成标准：
- 用户可以围绕“当前句”完成完整学习闭环
- 所有关键按钮启用状态符合预期
- 训练过程不依赖用户理解隐藏状态

---

## 阶段 E：日志与可见错误优化
目标：让行为可追踪，错误可理解。

步骤：

1. 为以下行为补日志：
   - 当前句切换
   - 上一句 / 下一句
   - 模式切换到具体模式名
   - 录音开始 / 停止
   - 覆盖旧录音
   - 播放原音
   - 播放录音
2. 区分普通提示和异常错误
3. 正常提示尽量更新 `hint_text`
4. 系统失败、文件失败、关键非法状态仍使用错误弹框

完成标准：
- `logs/app.log` 能清晰追踪主要学习流程
- 用户不会因为普通状态变化频繁看到弹窗
